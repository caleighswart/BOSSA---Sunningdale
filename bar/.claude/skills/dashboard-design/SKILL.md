# BOSSA Bar Dashboard — Design System

The dashboard at `https://bossa-sunningdale.netlify.app/` is generated by [bar/generate_dashboard.py](../../../generate_dashboard.py). It must feel like part of the BOSSA Goodtimes brand — vintage, hospitality-warm, not a generic SaaS table.

Source of truth for the brand: `bossagoodtimes.com` (vintage shield logo, dark hero photography, cream + yellow accents, custom slab/handwritten display font "Rakelsy"). The dashboard is internal but should still feel like *Bossa*.

## Design tokens

**Palette** — derive every color from this list. Do not introduce Tailwind grays, generic blues, or off-brand teals.

| Token | Hex | Use |
|---|---|---|
| `--bossa-ink` | `#1A1A1A` | Primary text, dark surfaces, hero band |
| `--bossa-charcoal` | `#33373D` | Secondary text, table borders |
| `--bossa-cream` | `#ECE6D6` | Page background, card surfaces |
| `--bossa-cream-deep` | `#E0D8C2` | Hover/striped rows, dividers |
| `--bossa-yellow` | `#EDCD45` | Primary accent — active tab, CTA, key numbers |
| `--bossa-yellow-deep` | `#D4B530` | Yellow hover/border |
| `--bossa-pink` | `#CC3366` | Secondary accent — variances, alerts that aren't "critical stock" |
| `--bossa-red` | `#B91C1C` | Critical stock only (semantic, not decorative) |
| `--bossa-amber` | `#B45309` | Low stock |
| `--bossa-green` | `#15803D` | Healthy stock |

The functional status colors (red/amber/green) are *muted, deeper* versions — not bright Tailwind defaults — so they sit on cream without screaming.

**Typography** — one font, used everywhere. The dashboard is a business tool first; readability and a professional feel beat brand-personality matching.

- **All text (hero, summary stats, tabs, tables, sections, body):** `"Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`.
- **Weights:** 400 body, 500 secondary text, 600 headings/key labels, 700 reserved for stat numerals and primary CTAs.
- **Numerals (SOH, par, value, stats):** add `font-variant-numeric: tabular-nums;` so columns align.
- **Do NOT use** Bebas Neue, Oswald, Anton, Playfair, or any condensed/slab/display font. The user has rejected display fonts on this surface — they read as heavy and unprofessional. Brand identity comes from the **palette and layout**, not typography.
- **Avoid** all-caps headings with wide letter-spacing — they felt cluttered. Use sentence-case or Title Case with tight tracking. Reserve uppercase only for tiny labels (≤11px) like stat captions or table column heads, with letter-spacing ≤ 0.06em.

Load Inter via Google Fonts `<link>` in the generated HTML head.

**Shape & spacing**
- Border radius: `12px` for cards, `8px` for buttons/chips, `4px` for table cells. Never `0` (too cold) and never `50%`/pill on data containers.
- Borders: 1px `--bossa-cream-deep` for table rows; 2px `--bossa-ink` for the hero band.
- Spacing: 24px between cards, 16px inside cards, 12px between table rows.
- Shadows: at most one — `0 1px 3px rgba(26,26,26,0.08)` on cards. No glowing or layered drop shadows.

## Layout principles

1. **Hero band, not a header bar.** Top of page is a dark `--bossa-ink` band ~120px tall containing the BOSSA logo/title in display font and the report date. This mirrors the dark hero on bossagoodtimes.com.
2. **Cream canvas.** Page background is `--bossa-cream`, not white. Cards are slightly lighter (`#F5F0E2`) or white with cream borders.
3. **Summary stat tiles in display font.** Critical / Low / Healthy / Total Value as four tiles at the top — large display-font numerals, small caps label below.
4. **Tabs as menu chips.** The tab strip (Critical / Low / Orders / All / Variances / Admin) should look like the menu category chips on bossa.co.za/sunningdale: 1px border, 8px radius, uppercase, generous horizontal padding (18-32px). Active tab = `--bossa-yellow` fill on `--bossa-ink` text.
5. **Tables breathe.** 12px row padding minimum, alternating row backgrounds (`--bossa-cream` / `#F5F0E2`), no vertical dividers.

## Components

- **Status pills** (Critical / Low / Healthy): solid background using semantic color, white text, uppercase, 8px radius, 4px 10px padding, display font, 11px size, letter-spacing 0.05em.
- **Fill bars** (% of par): 8px tall, 4px radius, `--bossa-cream-deep` track, semantic-colored fill. The `_pct_bar` helper in `generate_dashboard.py` already does this — keep the structure, just update colors.
- **Order cards** (in Orders tab): cream surface, supplier name in display font, contact line in small body text, "Place Order via WhatsApp" button uses `--bossa-ink` background with `--bossa-yellow` text (inverted) so it pops as the primary action.
- **Empty states**: lowercase italic body text, `--bossa-charcoal` color, never an icon-only "all clear" badge.

## Voice (microcopy)

The brand voice on bossagoodtimes.com is warm and informal ("Goodtimes", "Welcome To Sunningdale"). Mirror that in:
- Empty states: "Nothing to flag — looking good." (not "No data" or "All clear ✓")
- Section titles: sentence-case warm headings, not corporate ("Tonight's order list", not "ORDER MANAGEMENT")
- Date stamp: "Friday 8 May, 2026" not "2026-05-08T05:00:00Z"

Keep emoji usage zero in the HTML. The brand never uses them. The Telegram brief can keep its emoji — different surface.

## Don'ts

- ❌ No Bootstrap-style cards with white backgrounds and gray borders.
- ❌ No generic blue (`#0d6efd`, `#3b82f6`, etc.) anywhere.
- ❌ No emoji in the HTML output.
- ❌ No icon libraries (Font Awesome, Heroicons). The brand uses photography and typography, not iconography.
- ❌ No gradients (the brand uses flat fills + photography).
- ❌ No condensed/slab/display fonts (Bebas Neue, Oswald, Anton, Impact, Playfair). Inter only.
- ❌ No all-caps headings with > 0.06em letter-spacing — looks cluttered, not premium.
- ❌ No bright Tailwind status colors — use the muted versions in the palette table.
- ❌ No rounded-full / pill shapes on anything wider than ~80px.

## When applying this

1. Read [bar/generate_dashboard.py](../../../generate_dashboard.py) — the entire CSS lives in the `<style>` block inside the `_html_template` function (or equivalent).
2. Replace color literals (`#dc2626`, `#16a34a`, `#d97706`, etc.) with the palette tokens above. Define them as CSS custom properties on `:root`.
3. Add the Google Fonts `<link>` to the `<head>`.
4. Update the hero, tab strip, and stat tiles to match the layout principles.
5. Keep all data structures and Python logic identical — this is a CSS/HTML-shape change only.
6. Test by running `python generate_dashboard.py` from `bar/` and opening `docs/index.html` locally before committing.
